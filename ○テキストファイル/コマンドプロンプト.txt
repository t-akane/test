●オプション設定　
・必須入力 not null
空欄にできない

・主キー primary key　
重複を避けるindex付与、自動的にnotnull

・自動連番　auto_increment　
自動的に連番値が登録されてゆく
必ず整数のデータ型を宣言

・ユニークキー unique
重複するものが登録できなくなる
primaryとの違いはnullが登録できる点。
自動的にindexも付与される。

・索引付与　index
フルスキャンしなくても見つけられる仕組み
負担減、早くなった
大きなデータや、よくアクセスする値につけるとよい。

・キー　key
２つ使い方がある。
①primarykey
②index

create table lesson
 (ind int not null primary key auto_increment,
 name vachar(100) unique,
 point int,
 team enum('red','blue','green'),
 index(name),
 key point(point));

●並べ替え

昇順(order by)
select * from テーブル名 order by point;
降順(order by desc)
select * from テーブル名 order by point desc;
指定した件数だけ(limit)
select * from テーブル名 limit 3;
途中から指定した件数だけ
select * from テーブル名 limit 2,2;


●テーブル結合
種類
内部結合　INNER JOIN
外部結合　
　左外部　LEFT　OUTER　JOIN
　右外部　RIGHT　OUTER JOIN
　完全外部　FULL OUTER JOIN
交差結合　CROSS JOIN

内部結合
結合するテーブルの双方にてフィールドが一致するレコード
A∩B  ：A と B の両方に属する要素全体の集合（かつ，共通部分，積集合，intersection）
外部結合
一致するデータに加えて、どちらかのテーブルのみにしかないデータについても表示される。
A∪B  ：A と B の少なくとも一方に属する要素全体の集合（または，和集合，union）
